# 2026-03-03 병해충 섹션 누락 분석 (Newsis NISX20260228_0003530198, 영농신문 57763)

## 결론 요약
- 두 URL은 현재 코드 기준으로 `pest` 분류 신호가 충분하며, 단순 하드코딩 누락이 아니라 **수집 윈도우/쿼리 회수/섹션 간 이동/최종 선택/크로스데이 중복제거**가 연동된 파이프라인 이슈 후보군으로 봐야 한다.
- 특히 이 저장소 상태에서는 `recent_items`에 뉴시스 URL이 존재하지만 아카이브 HTML에는 해당 URL이 없어, **상태(state)와 렌더 결과 간 불일치**가 관측된다.

## 근거 1) 오늘 브리핑의 집계 윈도우
- 2026-03-03 아카이브는 `2026-02-27 07:00 ~ 2026-03-03 07:00` 윈도우를 사용한다.
- 따라서 2/28 기사라면 시간대 자체로는 포함 가능성이 있다(시간이 윈도우 내라면).

## 근거 2) 현재 결과물의 pest 섹션
- 2026-03-03 pest 섹션은 3건이며, 질의된 두 URL은 렌더 결과에 없다.

## 근거 3) 코드상으로 두 URL 유형은 pest로 가야 정상
- 테스트에서 Newsis `NISX20260228_0003530198`는 best section이 `pest`로 고정 검증된다.
- 테스트에서 Youngnong `idxno=57763`도 best section이 `pest`로 고정 검증된다.
- policy 게이트가 pest 실행형 기사(예찰/약제/전수조사)를 떨어뜨리지 않도록 별도 테스트가 있다.
- policy로 들어와도 글로벌 재분류/후처리에서 pest로 이동시키는 테스트가 있다.

## 근거 4) 파이프라인상 실제 누락 가능 지점(하드코딩 아님)
1. 수집 단계 시간 필터
   - `pubDate`가 윈도우 밖이면 즉시 제외된다.
2. 섹션별 relevance 게이트
   - 각 섹션 `is_relevant` 필터를 통과해야 후보가 유지된다.
3. 크로스데이 중복 제거
   - canonical URL/norm key가 최근 이력에 있으면 수집 단계에서 제외된다.
4. 전역 재분류 + 최종 선택
   - policy/other로 유입되어도 재분류되지만, 최종 선택 임계치/다양성/중복제거에서 탈락 가능.

## 근거 5) 상태-출력 불일치 관측
- `.agri_state.json`의 `recent_items`에 `https://newsis.com/view/NISX20260228_0003530198`가 존재한다.
- 하지만 동일 날짜 아카이브 HTML에는 해당 URL이 없다.
- 이는 “최종 렌더 산출물 기준 누락”을 판단할 때, state를 절대 근거로 쓰면 오판 가능함을 의미한다.
